021/**
022 * Provides the API for fetching and saving beans to a particular DataSource.
023 * <p>
024 * <b>Registration with the Ebean Singleton:</b><br/>
025 * When a EbeanServer is constructed it can be registered with the Ebean
026 * singleton (see {@link ServerConfig#setRegister(boolean)}). The Ebean
027 * singleton is essentially a map of EbeanServer's that have been registered
028 * with it. The EbeanServer can then be retrieved later via
029 * {@link Ebean#getServer(String)}.
030 * </p>
031 * <p>
032 * <b>The 'default' EbeanServer</b><br/>
033 * One EbeanServer can be designated as the 'default' or 'primary' EbeanServer
034 * (see {@link ServerConfig#setDefaultServer(boolean)}. Many methods on Ebean
035 * such as {@link Ebean#find(Class)} etc are actually just a convenient way to
036 * call methods on the 'default/primary' EbeanServer. This is handy for
037 * applications that use a single DataSource.
038 * <p>
039 * There is one EbeanServer per Database (javax.sql.DataSource). One EbeanServer
040 * is referred to as the <em>'default'</em> server and that is the one that
041 * Ebean methods such as {@link Ebean#find(Class)} use.
042 * </p>
043 * <p>
044 * <b>Constructing a EbeanServer</b><br/>
045 * EbeanServer's are constructed by the EbeanServerFactory. They can be created
046 * programmatically via {@link EbeanServerFactory#create(ServerConfig)} or they
047 * can be automatically constructed on demand using configuration information in
048 * the ebean.properties file.
049 * </p>
050 * <p>
051 * Example: Get a EbeanServer
052 * </p>
053 * <p>
054 * <pre>{@code
055 * // Get access to the Human Resources EbeanServer/Database
056 * EbeanServer hrServer = Ebean.getServer("HR");
057 *
058 *
059 * // fetch contact 3 from the HR database Contact contact =
060 * hrServer.find(Contact.class, new Integer(3));
061 *
062 * contact.setStatus("INACTIVE"); ...
063 *
064 * // save the contact back to the HR database hrServer.save(contact);
065 * }</pre>
066 * <p>
067 * <p>
068 * <b>EbeanServer has more API than Ebean</b><br/>
069 * EbeanServer provides additional API compared with Ebean. For example it
070 * provides more control over the use of Transactions that is not available in
071 * the Ebean API.
072 * </p>
073 * <p>
074 * <em>External Transactions:</em> If you wanted to use transactions created
075 * externally to eBean then EbeanServer provides additional methods where you
076 * can explicitly pass a transaction (that can be created externally).
077 * </p>
078 * <p>
079 * <em>Bypass ThreadLocal Mechanism:</em> If you want to bypass the built in
080 * ThreadLocal transaction management you can use the createTransaction()
081 * method. Example: a single thread requires more than one transaction.
082 * </p>
083 *
084 * @see Ebean
085 * @see EbeanServerFactory
086 * @see ServerConfig
087 */

477  /**
478   * Start a transaction with 'REQUIRED' semantics.
479   * <p>
480   * With REQUIRED semantics if an active transaction already exists that transaction will be used.
481   * </p>
482   * <p>
483   * The transaction is stored in a ThreadLocal variable and typically you only
484   * need to use the returned Transaction <em>IF</em> you wish to do things like
485   * use batch mode, change the transaction isolation level, use savepoints or
486   * log comments to the transaction log.
487   * </p>
488   * <p>
489   * Example of using a transaction to span multiple calls to find(), save()
490   * etc.
491   * </p>
492   * <p>
493   * <pre>{@code
494   *
495   *    // start a transaction (stored in a ThreadLocal)
496   *    ebeanServer.beginTransaction();
497   *    try {
498   *        Order order = ebeanServer.find(Order.class,10);
499   *
500   *        ebeanServer.save(order);
501   *
502   *        ebeanServer.commitTransaction();
503   *
504   *    } finally {
505   *        // rollback if we didn't commit
506   *        // i.e. an exception occurred before commitTransaction().
507   *        ebeanServer.endTransaction();
508   *    }
509   *
510   * }</pre>
511   * <p>
512   * <h3>Transaction options:</h3>
513   * <pre>{@code
514   *
515   *     Transaction txn = ebeanServer.beginTransaction();
516   *     try {
517   *       // explicitly turn on/off JDBC batch use
518   *       txn.setBatchMode(true);
519   *       txn.setBatchSize(50);
520   *
521   *       // control flushing when mixing save and queries
522   *       txn.setBatchFlushOnQuery(false);
523   *
524   *       // turn off persist cascade if needed
525   *       txn.setPersistCascade(false);
526   *
527   *       // for large batch insert processing when we do not
528   *       // ... need the generatedKeys, don't get them
529   *       txn.setBatchGetGeneratedKeys(false);
530   *
531   *       // explicitly flush the JDBC batch buffer
532   *       txn.flushBatch();
533   *
534   *       ...
535   *
536   *       txn.commit();
537   *
538   *    } finally {
539   *       // rollback if necessary
540   *       txn.end();
541   *    }
542   *
543   * }</pre>
544   * <p>
545   * <p>
546   * If you want to externalise the transaction management then you use
547   * createTransaction() and pass the transaction around to the various methods on
548   * EbeanServer yourself.
549   * </p>
550   */

558  /**
559   * Start a transaction typically specifying REQUIRES_NEW or REQUIRED semantics.
560   * <p>
561   * <p>
562   * Note that this provides an try finally alternative to using {@link #execute(TxScope, TxCallable)} or
563   * {@link #execute(TxScope, TxRunnable)}.
564   * </p>
565   * <p>
566   * <h3>REQUIRES_NEW example:</h3>
567   * <pre>{@code
568   * // Start a new transaction. If there is a current transaction
569   * // suspend it until this transaction ends
570   * Transaction txn = server.beginTransaction(TxScope.requiresNew());
571   * try {
572   *
573   *   ...
574   *
575   *   // commit the transaction
576   *   txn.commit();
577   *
578   * } finally {
579   *   // end this transaction which:
580   *   //  A) will rollback transaction if it has not been committed already
581   *   //  B) will restore a previously suspended transaction
582   *   txn.end();
583   * }
584   *
585   * }</pre>
586   * <p>
587   * <h3>REQUIRED example:</h3>
588   * <pre>{@code
589   *
590   * // start a new transaction if there is not a current transaction
591   * Transaction txn = server.beginTransaction(TxScope.required());
592   * try {
593   *
594   *   ...
595   *
596   *   // commit the transaction if it was created or
597   *   // do nothing if there was already a current transaction
598   *   txn.commit();
599   *
600   * } finally {
601   *   // end this transaction which will rollback the transaction
602   *   // if it was created for this try finally scope and has not
603   *   // already been committed
604   *   txn.end();
605   * }
606   *
607   * }</pre>
608   */
