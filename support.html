Getting help

For help post questions and issues to the Ebean google group.

Github Issues Policy

Quality Issues posted to Github of insufficient quality will be removed.
No protracted discussions Issues likely to result in protracted discussion must be posted to the Ebean google group.
No questions Do not post questions as Github issues. These will be removed and you will be asked to post questions to the Ebean google group instead.

Why this Policy exists

Avoid spamming Watchers

People register to watch the github issues register to keep track of changes to the project. As such protracted discussions effectively spam all these people watching the project and we don't want that.

Quality searching

We want to be able to search the github issues now and in the long term and we want those searches to result in high quality results. As such we want issues to relate closely to actual commits.

Future readers

We want to avoid low quality issues with protracted discussions as these become much more difficult to read and understand

Ebean google group

Post questions or issues to the Ebean google group. This is monitored by Ebean developers and it's most experienced users and here you will get the best help.

Stackoverflow

You can alternatively post questions on Stack Overflow tagged with ebean. Note that stackoverflow is more casually monitored due to time constraints. It is fine if you want to post first to stackoverflow and then if you don't get an answer to post onto the google group.

Github issues

For this project we want to use Github issues for Bugs, Enhancements and Refactoring changes. That is, things that mostly tie back to a specific commit change in code.

  • Complex issues must be discussed on Ebean google group first to avoid spamming watchers.
  • Questions posted to github will be closed and content cleared.
  • Low quality issues posted to github will be closed and content cleared.

In short, always post to Ebean google group first.
