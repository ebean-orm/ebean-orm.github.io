Maven

Ebean main artifact

Core artifact of Ebean ORM

<dependency>
  <groupId>org.avaje.ebeanorm</groupId>
  <artifactId>avaje-ebeanorm</artifactId>
  <version>4.0.4</version>
</dependency>

Agent

Enhancement agent for Ebean ORM

<dependency>
  <groupId>org.avaje.ebeanorm</groupId>
  <artifactId>avaje-ebeanorm-agent</artifactId>
  <version>4.1.2</version>
</dependency>

Maven Enhancement

Maven Enhancement plugin

<plugin>
  <groupId>org.avaje.ebeanorm</groupId>
  <artifactId>avaje-ebeanorm-mavenenhancer</artifactId>
  <version>4.1.2</version>
  <executions>
     <execution>
      <id>main</id>
      <phase>process-classes</phase>
      <configuration>
        <classSource>target/classes</classSource>
        <packages>org.example.domain.**</packages>
        <transformArgs>debug=1</transformArgs>
      </configuration>
      <goals>
        <goal>enhance</goal>
      </goals>
    </execution>
  </executions>
</plugin>

Releases

4.0.4 / 29 May 2014

  • API - Model and Finder changes
  • BUG - Issue with lazy loading after a findIterate query
  • BUG - transaction.setBatchFlushOnQuery(false) not working for some queries
  • ENH - Minor - not log extra SQL into log when using jdbc batch mode

4.0.3 / 27 May 2014

  • API - Add Play Model and Finder to Ebean
  • BUG - Memory issue (Hard refs) when using findIterate
  • BUG - Backport to JDK 6 (aka fix for issue #129 in 4.x)
  • ENH - Add support for mocking EbeanServer in the Ebean singleton (to use Mockito or similar)

4.0.2 / 18 May 2014

  • API - Deprecating findPagingList(), will provide a replacement without fetch ahead etc
  • API - Deprecating - Query findFutureList() and SqlQuery findFutureList()
  • API - Breaking API Change - Stateless updates no longer need explicit Set updateProps.
  • API - Breaking API Change - remove BeanState.setReference()
  • API - Removed Feature - Remove support for multiple @Id properties (use EmbeddedId instead). Remove support for saving an entity bean that does not have an @Id property
  • API - Breaking API Change - BeanPersistRequest.getOldValues() replaced with ... Map<String,ValuePair> getUpdatedValues()
  • API - Breaking API Change - Remove Query setBackgroundFetchAfter() feature
  • API - Breaking API Change - Remove deprecated QueryListener. You must convert to findIterate() or findVisit()
  • BUG - Exception using findIds() with inheritance SINGLE_TABLE
  • BUG - L2 Cache not working for embedded beans
  • BUG - On save @OneToOne does not cascade parent IDs to child node
  • BUG - Ebean doesn't honor @Transient for non-primitive array properties
  • BUG - DDL creation doesn't handle nullable attribute of @JoinColumn
  • BUG - When using @Transient along with @Formula I'm not seeing any values in the field.
  • BUG - Invalid query when ORDER BY with DISTINCT
  • BUG - Stateless update cascading to OneToOne or ManyToOne incorrectly tries to INSERT rather than UPDATE
  • BUG - @AttributeOverrides annotation for attribute definition in entity does not work for embedded primary keys
  • BUG - Embedded Entities with Autofetch Exception
  • BUG - Using findIterate with MySQL streams the entire result set at once
  • BUG - issue when deleting an Entity with relationship which is using @EmbeddedId as a composite keys
  • BUG - disjunction expression should not produce inner join - left outer join instead
  • BUG - Stateless recursive updates for entities with version field will create Inserts instead of Updates
  • BUG - Many relations are deleted when accessing many field in BeanPersistAdapter on stateless updates bug
  • ENH - Ability to store UUID as binary(16) rather than varchar(40)
  • ENH - Allow the global row limit for queries to be configurable ( CQuery.GLOBAL_ROW_LIMIT)
  • ENH - Add support for @PrePersist, @PostPersist, @PostLoad,...
  • ENH - RawSql to support discriminator column mapping. enhancement
  • ENH - Additional API - added update(Collection beans); and insert(Collection beans); to EbeanServer
  • ENH - Enhancement changed - Must use 4.1.0 of agent and mavenenhancer etc
